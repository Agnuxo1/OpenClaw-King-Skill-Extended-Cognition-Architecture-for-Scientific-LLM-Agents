ABSTRACT
We present King-Skill, a master cognitive orchestration layer for Large Language Model (LLM) agents that applies the principle of extended cognition (Clark & Chalmers, 1998) to transform how agents consume tokens. Instead of using language model reasoning tokens to execute deterministic computations—numerical calculations, graph analysis, document conversion, literature retrieval—King-Skill routes these tasks to verified specialist tools and synthesises results with minimal output tokens. A complementary token-compression skill (v3) replaces verbose natural language output with compact mathematical notation, Python pseudocode, and formal logic symbols wherever these are semantically equivalent.
The system comprises 21 skills spanning scientific computing, formal verification, data retrieval, and document generation. We report empirical test results from a suite of 52 automated tests (52/53 passing; 1 failure isolated and documented) and a routing accuracy evaluation on 19 handcrafted cases (18/19 correct). Token reduction estimates of > 60% are based on single-point measurements and are NOT yet statistically validated. Claims about reasoning quality preservation are theoretical and require controlled study.
Data transparency key:   [EMPIRICAL]  Measured in controlled tests.   [ESTIMATE]  Single-point or informal measure.   [THEORETICAL]  Design intent, not yet measured.


§1  Introduction
Large Language Models consume tokens proportionally to the complexity of the reasoning they perform. When an LLM calculates eigenvalues, sorts datasets, converts documents, or searches literature using only language tokens, it expends compute on tasks that deterministic tools could execute in milliseconds. The resulting inefficiency is not merely economic—it degrades response quality by occupying context space that could hold genuine reasoning.
King-Skill reframes the LLM as an orchestrator rather than an executor, drawing on the philosophical framework of extended cognition:
"The Extended Mind" — Clark & Chalmers (1998):
If a part of the world functions as a process that, if it were done in the head, we would have no hesitation in recognising as part of the cognitive process—then that part of the world is part of the cognitive process. A mathematician's whiteboard is not external to their thought; it is part of it.
By the same logic: the SAT solver, the Python executor, the arXiv API—these are not external tools the LLM "uses". They are part of the LLM's cognitive system when properly integrated.

§2  System Architecture
The architecture is a three-tier hierarchy: a master router (King-Skill) at priority 0 that classifies every incoming task and dispatches to one of 20 domain skills, with token-compression always active on all output.

DISPATCH ALGORITHM
PYTHONdef king_dispatch(task: Task) -> Response:    # Step 1: classify (< 30 tokens)    category = classify(task)    # Categories: COMPUTE | SIMULATE | VERIFY | RETRIEVE | TRANSFORM | GRAPH | REASON    # Step 2: lookup skill in registry    skill = DISPATCH_TABLE.get(category)    if skill is None:        # No tool exists → reason directly + apply token-compression        return reason_compressed(task)    # Step 3: delegated execution    try:        result = execute_skill(skill, task)        return synthesize_compressed(result, task.context)  # minimal tokens    except ToolError as e:        return fallback(task, e)

§3  Token-Compression Skill v3
The token-compression skill is permanently active on all King-Skill output. It enforces a strict separation between two independent token budgets:
Thinking budget (chain-of-thought): free — natural language + mathematics + code. DO NOT COMPRESS. Intermediate tokens are compute; compressing them degrades reasoning quality (Wei et al., 2022).  [THEORETICAL]
Output budget: maximum compression via mathematical notation, Python pseudocode, Lean 4 formal logic, and emoji dictionary. The target reader (Francisco Angulo de Lafuente) is an expert who reads dense notation without loss.  [ESTIMATE]

KNOWN FAILURE — SKILL-09 SYMPY (DOCUMENTED)
BUG REPORT — HONEST DISCLOSURE
Skill-09 (sympy-algebra) received an algebraic task that matched the "symbolic_algebra" dispatch key. The dispatch routing correctly identified the category, but the routing test was written with "sympy" as a keyword and the King-Skill v2.0 dispatch table maps "symbolic_algebra" to "skill-09-sympy". The test reveals a keyword-vs-category mismatch in the routing logic — the task was correctly caught by the python-executor fallback, but the primary dispatch returned the wrong skill first.
This bug is logged and scheduled for fix in v2.1. The fallback mechanism prevented any actual incorrect execution. [EMPIRICAL]


§6  Related Work
King-Skill builds on and synthesises several threads of recent LLM research:
CHAIN-OF-THOUGHT PROMPTING (WEI ET AL., 2022) [REF 2]
Wei et al. demonstrate that providing intermediate reasoning steps ("chain of thought") significantly improves LLM performance on multi-step reasoning tasks. King-Skill respects this finding explicitly: the thinking budget is free and uncompressed, preserving full CoT. The innovation is to compress only the output budget—the tokens the user or downstream system receives—rather than the internal reasoning chain. [THEORETICAL basis for the two-budget separation]
PROGRAM OF THOUGHTS (CHEN ET AL., 2022) [REF 3]
Chen et al. show that using code (rather than natural language) as the reasoning medium—"Program of Thought"—outperforms standard CoT on numerical and algorithmic tasks. This directly validates our skill-01 (python-executor) design: for algorithmic problems, code is both the reasoning medium and the execution artefact. King-Skill extends this insight system-wide. [EMPIRICAL in Chen et al.; our application is THEORETICAL]
TOOLFORMER (SCHICK ET AL., 2023) [REF 5]
Schick et al. demonstrate that LLMs can learn to use external APIs and tools as part of their reasoning process through self-supervised training. King-Skill extends this with a structured priority hierarchy, explicit token compression, formal verification, and specialisation for scientific computing contexts.
REACT (YAO ET AL., 2022) [REF 6]
Yao et al. propose the Reason + Act framework where LLMs interleave reasoning traces with tool actions. King-Skill shares the core insight but differs in placing the routing decision before reasoning (not interleaved), applying compression to all output, and targeting a specific scientific computing workflow.

OPEN RESEARCH QUESTIONS
Q1: Does compressing output tokens degrade internal reasoning quality?
→ Requires access to thinking token counts + quality evaluation. Wei et al. (2022) motivates the hypothesis that thinking-token compression degrades quality, but the effect on output-only compression is unknown. [THEORETICAL]
Q2: What is empirical routing accuracy on a stratified corpus of 1000+ tasks?
→ 18/19 on 19 handcrafted cases is insufficient for publication-quality evaluation. A stratified sampling of real user tasks is required. [ESTIMATE that 94.7% generalises is unsupported]
Q3: Can pause_tokens (Goyal et al., 2023) reduce thinking cost without quality loss?
→ Architectural change; not testable with current LLM APIs. Would require model fine-tuning. [THEORETICAL]
Q4: Does Lean 4 verification of CBM claims improve OpenCLAW peer-review consensus scores?
→ P2PCLAW (p2pclaw.com) is the natural testbed for this question. Requires completing the Lean 4 proof of consensus_convergence. [THEORETICAL]
