Certification Work Guide for King-Skill v4

A complete operational protocol for replacing every estimated or theoretical claim in v3 with empirically-validated measurements — covering compression ratios, routing accuracy, reasoning quality, cost data, and reproducibility artefacts

📅 Based on v3 gap analysis · April 2026 🎯 Target: King-Skill v4 with zero unvalidated claims ⏱ Estimated execution time: 6–8 weeks 👥 Requires: Silicon agent + Carbon human coordinator

Overview — What v4 Must Prove

King-Skill v3 is an honest prototype description. Every quantitative claim is either estimated (small samples) or theoretical (no empirical test). The v4 paper requires replacing all estimates with measurements from controlled experiments. This guide enumerates every experiment, specifies its measurement protocol, defines the statistical acceptance criteria, and lists the output artefact that goes into the v4 paper.

The guide is organised into six phases. Phases 1 and 2 are prerequisites — no later phase produces valid data without them. Phases 3–6 can run in parallel once Phase 1 infrastructure is complete.

EXP-04 Corpus Compression Study — Real P2PCLAW Drafts P1 Critical TODO

Goal: Measure mean ± SD token compression ratio on a random sample of real P2PCLAW paper drafts, not hand-crafted examples. Replace the "~9% mixed, ~33% STEM" estimates with validated figures.

Corpus sizen ≥ 200 paragraph pairs (natural → compressed) from ≥ 20 real P2PCLAW papers across 5 domains
Domain stratificationThermodynamics (20%), quantum mechanics (20%), distributed systems (20%), ML (20%), synthetic biology (20%)
Tokenizertiktoken cl100k_base AND o200k_base (GPT-4o) — report both; compare cross-model
Compression procedureClaude Sonnet 4.6 applies compression layer to each paragraph; record token counts before and after
StatisticsMean, SD, 95% CI (bootstrap n=10,000), median, p5/p95 percentiles; stratified by domain and content type
Content type labelsFor each paragraph: equation-heavy / reaction-heavy / prose-heavy / mixed. Compare savings per label.
Acceptance criterionn ≥ 200 pairs; 95% CI width ≤ 5 percentage points for primary STEM mean; all domain strata represented
Output artefactdata/compression_corpus_v4.csv, analysis/compression_stats.py, figures for paper

Measurement Script Skeleton

import tiktoken, anthropic, pandas as pd, json from scipy import stats enc = tiktoken.get_encoding("cl100k_base") client = anthropic.Anthropic() def compress_paragraph(text: str) -> str: """Apply King-Skill compression layer to a paragraph.""" r = client.messages.create( model="claude-sonnet-4-6", max_tokens=2000, system="Apply formal notation compression: replace verbose \ descriptions with math, chemical formulas, and equations \ where appropriate. Output only the compressed text.", messages=[{"role": "user", "content": text}] ) return r.content[0].text def measure_pair(natural: str, domain: str, content_type: str) -> dict: compressed = compress_paragraph(natural) t_nat = len(enc.encode(natural)) t_comp = len(enc.encode(compressed)) savings_pct = (t_nat - t_comp) / t_nat * 100 return { "domain": domain, "content_type": content_type, "tokens_natural": t_nat, "tokens_compressed": t_comp, "savings_pct": savings_pct, "natural": natural, "compressed": compressed } # Run on corpus, collect results DataFrame # df.to_csv("data/compression_corpus_v4.csv") # Report: mean±SD, 95% CI bootstrap, by domain+content_type
Required v4 Figures

Fig A: Compression ratio distribution (violin plot, by domain). Fig B: Mean ± 95% CI, stratified by content type (equation-heavy, reaction-heavy, prose-heavy, mixed). Fig C: Scatter plot tokens_natural vs. savings_pct to show heteroscedasticity.

EXP-08 500+ Task Routing Accuracy Study P1 Critical TODO

Goal: Measure routing accuracy with a confidence interval that actually supports the stated percentage. At n=500 with p≈0.95, the 95% CI has width ±1.9 percentage points — publishable precision.

Corpus sizen = 500 tasks minimum. Stratified: ~25 per skill × 20 skills = 500 core, plus 50 ambiguous + 50 fallback = 600 total
Task generationSilicon agent generates 5 task variants per skill per difficulty level (easy/medium/hard/boundary). Carbon reviews and corrects ground truth labels.
Inter-rater reliabilityCarbon labels 100 tasks independently; compute Cohen's κ with Silicon's labels. κ ≥ 0.80 required before proceeding.
MeasurementFor each task: predicted_skill, correct_skill, is_correct (bool), confidence_score (if available)
StatisticsOverall accuracy + 95% Wilson CI; per-skill precision/recall/F1; confusion matrix; accuracy by difficulty level
Acceptance criterionn ≥ 500 labelled tasks; Cohen's κ ≥ 0.80; all 20 skills represented with ≥ 20 tasks each
Output artefactdata/routing_tasks_v4_500.json, analysis/routing_accuracy.py, confusion matrix figure
Required sample size derivation: Target: CI width ≤ 4 pp at 95% confidence For p ≈ 0.95: n ≥ z²·p·(1−p) / (E/2)² n ≥ 1.96² × 0.95 × 0.05 / 0.02² n ≥ 456 → use n = 500 (adds 10% margin) Per-skill precision/recall at n=25/skill: 95% CI width ≈ ±20 pp (adequate for directional inference) For ±10 pp: need n≥100/skill → 2,000 total (stretch goal)

Phase 4 · Controlled Baseline Experiment

The most critical experiment in this guide. Without a controlled comparison of King-Skill vs. an unrouted LLM, the paper cannot claim that the architecture provides any benefit. This phase executes the experiment designed in v3 §7.1.
⏱ Duration: 3–4 weeks (includes judge evaluation time) 👤 Owner: Silicon (paper generation) + Carbon (judge coordination) 🔒 Requires: Phase 1 complete; Phase 2 EXP-06 (cost baseline) 📦 Outputs: baseline_results.csv, judge_scores.csv, statistical_report.pdf
EXP-11 King-Skill vs. Plain LLM — Controlled A/B P1 Critical TODO

Goal: Produce paired scientific papers under two conditions: (A) King-Skill agent with full routing and compression, (B) same base LLM with no routing and no compression layer. Have both evaluated by the 17-judge panel. Compare consensus scores.

Base LLMClaude Sonnet 4.6 for both conditions. Same system prompt, same input task.
Task pooln = 50 P2PCLAW-style research tasks. Stratified: 10 per domain (thermodynamics, quantum, distributed systems, ML, synthetic biology)
Condition AKing-Skill: king-dispatch active, all 20 skills available, compression layer active
Condition BBaseline: same prompt, no dispatch, no skills, no compression. Direct LLM generation only.
BlindingJudges receive papers without condition labels. Randomise order. Judges cannot identify which condition produced which paper.
Primary outcome17-judge consensus score (0–1). Paired t-test or Wilcoxon signed-rank. α=0.05, two-tailed.
Secondary outcomesOutput token count, API cost, proportion meeting 2,500-token minimum, factual error rate (EXP-16)
Power analysisn=50 pairs, assuming σ=0.10, minimum detectable effect Δ=0.05. Power = 0.80 at α=0.05. (Increase n if σ turns out larger.)
Acceptance criterionn=50 complete pairs; all tasks evaluated by ≥14/17 judges; Krippendorff's α ≥ 0.40
Output artefactdata/baseline_experiment_v4.csv, statistical test results, forest plot of per-domain effects
ANALYSIS PLAN (pre-registered): Primary test: H₀: μ_KingSkill = μ_Baseline (no difference in judge scores) H₁: μ_KingSkill ≠ μ_Baseline (two-tailed) Test: Wilcoxon signed-rank (paired, non-parametric) α = 0.05, power = 0.80 Secondary tests (Bonferroni corrected, α' = 0.05/5 = 0.01): H₁a: Score(A) > Score(B) on scientific accuracy dimension H₁b: Score(A) > Score(B) on methodological rigour dimension H₁c: No significant difference on clarity dimension (non-inferiority) H₁d: Cost(A) vs Cost(B): expect A higher due to tool overhead H₁e: Error_rate(A) < Error_rate(B) for computation-heavy tasks Expected pattern (hypothesis): King-Skill advantage: tasks requiring computation (numerical accuracy) No difference: writing-intensive tasks with no deterministic computation Possible disadvantage: compressed output scoring lower on clarity

Critical — pre-registration: This analysis plan must be published (e.g., on OSF or as a GitHub commit) before running the experiment. Post-hoc analysis plan adjustments invalidate the statistical claims. Carbon must complete this before Silicon begins generating papers.

EXP-13 Inter-Rater Agreement — Krippendorff's α P2 High TODO

Goal: Compute Krippendorff's α across all 17 judges on the EXP-12 data. Report per-dimension and overall α with bootstrap confidence intervals.

MetricKrippendorff's α (interval scale), bootstrap 95% CI (n=5,000 resamplings)
Acceptanceα ≥ 0.40 (moderate agreement) on overall score. α ≥ 0.30 minimum per dimension.
InterpretationIf α < 0.40: the panel is too heterogeneous to aggregate into a single score. Explore clustering judges by tier and reporting tier-level scores separately.
Librarykrippendorff Python package (pip install krippendorff)

import krippendorff, numpy as np # scores: 2D array shape (n_judges, n_papers) alpha = krippendorff.alpha( reliability_data=scores, level_of_measurement="interval" ) # Bootstrap CI boot_alphas = [] for _ in range(5000): idx = np.random.choice(scores.shape[1], scores.shape[1]) boot_alphas.append(krippendorff.alpha(scores[:, idx], "interval")) ci_low, ci_high = np.percentile(boot_alphas, [2.5, 97.5]) print(f"α = {alpha:.3f} 95% CI [{ci_low:.3f}, {ci_high:.3f}]")
